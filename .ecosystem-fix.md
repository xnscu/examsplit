# 修复说明：ecosystem.config.js → ecosystem.config.cjs

## 问题
原错误信息：
```
ReferenceError: module is not defined in ES module scope
This file is being treated as an ES module because it has a '.js' file extension
and '/root/examsplit/package.json' contains "type": "module".
```

## 原因
- 项目的 `package.json` 设置了 `"type": "module"`，表示使用 ES Module 模式
- PM2 的配置文件使用了 CommonJS 语法（`module.exports`）
- 这两者产生了冲突

## 解决方案
将 `ecosystem.config.js` 重命名为 `ecosystem.config.cjs`

- `.cjs` 扩展名明确告诉 Node.js 这是 CommonJS 格式
- PM2 可以正常读取 `.cjs` 配置文件

## 已更新的文件
1. `ecosystem.config.js` → `ecosystem.config.cjs` (重命名)
2. `package.json` - 更新所有 PM2 脚本引用
3. `pm2-commands.sh` - 更新所有配置文件引用
4. `PM2-USAGE.md` - 更新文档
5. `QUICK-START-PM2.md` - 更新快速入门指南

## 验证
```bash
pm2 start ecosystem.config.cjs --only exam-batch
```

✅ 现在可以正常工作了！
